# stellar-voyage 設計書

作成日: 2026-07-05
状態: ユーザー承認済み（実装計画作成前）

## 概要

光速・超光速で宇宙を旅し、実在の星から人類が到達できない遠方の生成宇宙までを探検できる
リアル志向の宇宙旅行シミュレーションゲーム。訪れた星の詳細情報を閲覧でき、発見記録が
図鑑としてローカルに蓄積される。

## 決定事項（要件ヒアリング結果）

| 項目 | 決定 |
|---|---|
| プラットフォーム | Web ブラウザ（Vite + TypeScript + Three.js、Cloudflare Pages 公開） |
| 宇宙データ | 実データ + 手続き生成のハイブリッド（Space Engine 方式） |
| 光速表現 | 亜光速で相対論演出あり、光速超えはワープ航法として別演出 |
| ゲーム性 | 探検 + 発見記録（燃料等の制約なし） |
| 接近レベル | 惑星を間近で眺める（着陸はしない。第 2 弾候補） |
| 描画エンジン | Three.js + 自作 GLSL シェーダー（WebGL2） |

## 1. 全体構成

- Vite + TypeScript + Three.js の静的 Web アプリ。サーバー不要。
- デプロイは Cloudflare Pages（orbit-tracker と同じ運用パターン）。
- 星カタログデータはビルド時にバイナリ同梱。発見記録はブラウザ IndexedDB。
- テストは vitest（orbit-tracker / realtime-earth と同じ）。

## 2. 宇宙データ

### 2.1 実データ層

- **HYG 星カタログ v3**（約 119,000 星）を使用。ライセンスは CC BY-SA 4.0 —
  クレジット表記を About / README に必ず記載する。
- ビルド時の前処理スクリプトで CSV → バイナリ（Float32Array 列指向、数 MB）に変換。
  含める列: 銀河座標 (x,y,z)、視等級、絶対等級、色指数 (B-V)、スペクトル型、固有名。
- **NASA Exoplanet Archive**（確認済み系外惑星、約 5,800 恒星系）を星名マッチングで
  HYG 星に付与。NASA データは自由利用可（出典表記推奨）。
- これによりシリウス・ベテルギウス・プロキシマケンタウルス b 等の実在天体に行ける。

### 2.2 手続き生成層

- カタログ範囲（太陽から数千光年）の外側は、天の川銀河の構造モデル
  （渦状腕・バルジ・薄い円盤・ハローの密度分布関数）に従って生成。
- 空間を階層グリッド（オクツリー的セル）に分割し、セル座標から導出する
  **決定論的シード**で星を生成。同じ場所に戻れば必ず同じ星が存在する
  （発見図鑑の同一性の前提）。
- 恒星の生成は実際の天文統計に従う:
  - 質量: 初期質量関数（Kroupa IMF 近似）
  - スペクトル型 O/B/A/F/G/K/M と温度・半径・光度: 主系列関係式 + 巨星/白色矮星の混入率
  - 色: 温度 → 黒体放射色変換
  - 惑星: スペクトル型別の出現率統計でシステム生成（軌道半径、惑星型: ガス/氷/岩石/海洋、
    ハビタブルゾーン判定）
- 銀河系外は遠方銀河（ポイント + ビルボード）として描画。他銀河へは
  ハイパージャンプでのみ到達でき、到達先は同アルゴリズム（別シード）で生成された
  銀河として探検できる。アンドロメダ等の近傍実在銀河は位置・名称のみ実データを使う。

### 2.3 星情報パネルの表示項目

名称/カタログ符号（生成星は座標由来の符号）、スペクトル型、表面温度、質量（太陽質量比）、
半径（太陽半径比）、光度（太陽光度比）、太陽からの距離（光年）、惑星一覧
（軌道長半径、推定型、ハビタブルゾーン内フラグ）、実在星は「実在」バッジ。

## 3. スケール問題の解法

惑星（10^7 m）から銀河間（10^22 m）まで約 15 桁超のスケール差を扱う。

- **カメラ相対描画（floating origin）**: 世界座標は CPU 側で float64（JS number）保持。
  GPU へはカメラからの相対座標のみ渡し、遠距離での float32 精度崩れ（ジッター）を回避。
- **対数深度バッファ**（Three.js `logarithmicDepthBuffer`）で近接惑星と遠方星野の同時描画。
- 描画レイヤー構造:
  1. 遠方銀河スカイボックス（背景）
  2. 星野: カタログ星 + 生成星をポイントクラウド描画（等級→サイズ/輝度、色指数→色）
  3. 近傍恒星系: 接近した恒星系のみ 3D 天体としてインスタンス化
  4. 惑星クローズアップ: 手続きテクスチャ（ガス惑星の縞、岩石惑星のクレーター、
     海洋惑星の海と雲）をシェーダー生成
- 恒星系突入時はローカル座標系（AU 単位）に切替える階層座標システム。

## 4. 移動と相対論演出

| モード | 速度域 | 演出・物理 |
|---|---|---|
| 通常航行 | 0〜0.999c | 相対論的視覚効果（下記）。HUD に γ と船内/地球時間 |
| ワープ | 〜毎秒数千光年（≈10¹¹ c） | 相対論効果を無効化しワープトンネル演出。銀河内横断（10 万光年）が数十秒〜数分 |
| ハイパージャンプ | 即時 | 銀河マップ / 図鑑から目的地選択してジャンプ。他銀河への到達はこのモードのみ |

### 相対論的視覚効果（通常航行時、GLSL 実装）

- **光行差**: 速度 β に応じて星の見かけ位置が進行方向に集中する（頂点シェーダーで
  方向ベクトルをローレンツ変換）。
- **相対論的ドップラー効果**: 前方の星が青方偏移、後方が赤方偏移。星の黒体温度を
  ドップラー係数でシフトして色変換。
- **ビーミング（前灯効果）**: 前方の star の見かけ輝度が増す。
- **時間の遅れ（浦島効果）**: HUD に船内経過時間と地球経過時間を並記。
  γ = 1/√(1-β²) を正式な式で計算し、加速履歴から積分。
- 物理式は特殊相対論の正式な式を使い、単体テストで検証する。

### 操作

- マウス: 視点操作（pointer lock またはドラッグ）
- W/S: スロットル増減（対数スケール、0 → km/s → 0.999c → ワープ域へ連続的に遷移）
- クリック: 天体選択（レイキャスト）
- スマホ: ドラッグ視点 + ピンチ/ボタンでスロットルの最小対応

## 5. UI

- HUD: 現在速度（km/s および c 単位）、γ 係数、船内時間/地球経過時間、
  選択目標の名称と距離。
- 銀河マップ: 俯瞰 2D/3D マップ。クリックでワープ先指定。現在地と訪問済み星を表示。
- 恒星系マップ: 突入中の系の軌道図。
- 発見図鑑: 訪問した星を自動記録（IndexedDB）。初到達日時・星の諸元・実在/生成の別。
  一覧から再ジャンプ可能。
- UI 言語は日本語。数値は SI + 天文単位（ly、AU、太陽質量比等）。

## 6. エラー処理

- WebGL2 非対応ブラウザ: 日本語の案内表示（対応ブラウザ一覧）。
- カタログデータ読込失敗: リトライ + 手続き生成のみモードへフォールバック。
- IndexedDB 不可（プライベートモード等）: 図鑑をメモリ動作にし、保存されない旨を表示。
- WebGL コンテキストロスト: 自動復帰処理。

## 7. テスト方針

- vitest 単体テスト:
  - 手続き生成の決定論性（同シード → 同一の星・惑星系）
  - 相対論計算（γ、ドップラー係数、光行差角）を解析解と照合
  - カタログ変換パイプライン（CSV → バイナリの整合性）
  - 座標系変換（銀河座標 ↔ ローカル系）
- 描画はスクリーンショットベースの目視確認（実装中に随時）。

## 8. マイルストーン

1. **M1 — 星空と飛行**: HYG カタログの星野描画 + 自由飛行 + ワープ + 星選択と情報パネル。
   この時点で「実在の星に行ける」体験が成立。
2. **M2 — 銀河全域と惑星**: 手続き生成で銀河全域 + 恒星系突入 + 惑星クローズアップ。
   銀河生成器は他銀河（別シード）にも流用できる構造にする。
3. **M3 — 相対論演出**: 光行差・ドップラー・ビーミング・時間遅れ HUD。
4. **M4 — 図鑑と公開**: 発見図鑑 + 銀河マップ + スマホ最小対応 + Cloudflare Pages 公開。
