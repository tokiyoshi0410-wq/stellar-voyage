# stellar-voyage 連続ズーム航法 UX 再設計 設計書

作成日: 2026-07-05
状態: ユーザー承認済み（実装計画作成前）
前提: M1（galaxy 星野+飛行）/ M2a（恒星系突入+惑星+NASA 実在データ）完成済み

## 概要

現行の「galaxy / system の 2 モード + ボタン切替」を、**単一の連続ズーム航法**へ作り替える。
起動時は本物の太陽系を斜め上から見下ろす宇宙船視点。マウスドラッグで視点周回、ホイールで
ズーム、WASD で全方向移動、画面のスピードスライダーで速度を自由調整。ズームアウトすると
太陽が銀河の一星になり、他の星へ飛んでズームインするとその星系が展開する。継ぎ目のない
直感的な操作を目標とする。

## 決定事項（ヒアリング結果）

| 項目 | 決定 |
|---|---|
| 操作モデル | マウスドラッグ（視点周回）+ ホイール（ズーム）+ WASD（移動）+ 速度スライダー |
| 太陽系 | 本物の 8 惑星（水金地火木土天海）をハードコード（実軌道・相対サイズ・型別色・土星の環） |
| 外への繋がり | ズームで銀河へ連続遷移（太陽 ⇄ 銀河星野 ⇄ 他星系がシームレス） |
| 開始画面 | 太陽系を斜め見下ろし |
| スコープ | 操作系とスケール遷移のみ作り替え。M1/M2a の資産（カタログ・NASA・惑星描画）は温存 |

## 1. 座標・スケール統一モデル（技術の核）

現行の「galaxy=パーセク+floating origin」「system=AU+直接カメラ」の二本立てを、
**単一の AU 単位ワールド + floating origin + 対数深度**に統一する（M1 の実証済み手法の拡張）。

- ワールド単位は **AU**。銀河の星は `starPos_pc × 206264.8` で AU 化して配置。
- カメラ世界位置は float64（`FloatingOrigin` 流用）、GPU へはカメラ相対の float32 を渡す。
  対数深度バッファ（既存 `Renderer`）で 0.05 AU（惑星至近）〜 数万パーセク（≈10¹⁰ AU）を同時描画。
- これにより惑星（〜1 AU）と銀河星野（〜10⁶ AU）が 1 シーン・1 カメラで共存し、モード分岐が消える。

## 2. カメラ・操作（NavigationController）

カメラは常に **フォーカス点**（現在の注目天体）を周回する。

- `focusStarIndex`: フォーカス星のカタログ index（起動時 0 = 太陽）。フォーカス星のワールド AU 位置を保持。
- `orbitYaw` / `orbitPitch`: フォーカス周りのカメラ角度。`viewDistanceAu`: フォーカスからのカメラ距離。
- 操作割り当て:
  - **ドラッグ**: `orbitYaw`/`orbitPitch` を変更（フォーカスを中心に視点周回、見下ろし角も自由）。
  - **ホイール**: `viewDistanceAu` を対数的に増減（ズーム。スケール遷移を駆動）。
  - **WASD**: フォーカス点をカメラ相対で平行移動（前後左右、= 空間を飛ぶ）。移動速度は速度スライダー値。
  - **速度スライダー**（画面下 DOM）: WASD 移動速度を対数スケールで連続調整（系内 〜 AU/秒、星間 〜 光年/秒）。
- 起動時: `focus = 太陽`, `viewDistanceAu ≈ 40`（太陽系全景）, `orbitPitch ≈ -35°`（斜め見下ろし）。

## 3. レイヤーフェード（連続スケール遷移）

系ディテールと銀河星野を 1 シーンに重ね、`viewDistanceAu` に応じてクロスフェードで継ぎ目を隠す。

- **系レイヤー**（フォーカス星の恒星球 + 惑星 + 軌道リング、`SystemScene` 流用）:
  `viewDistanceAu` が小さいほど不透明。`fadeSystem = 1 - smoothstep(300, 30000, viewDistanceAu)`
  （≈300 AU 以下で完全表示、≈30000 AU ≈ 0.15 pc で消える）。0 のときは系オブジェクトを非表示。
- **銀河星野レイヤー**（全カタログ星をポイント描画、`StarField` 流用）: 常時描画。ただしフォーカス星
  自身のポイントは系表示中は隠す（二重描画回避）。遠ざかるほど銀河が主役になる。
- 惑星・恒星球のマテリアル透明度に `fadeSystem` を乗算。ポイント星は距離で自然に小さくなるため
  追加フェード不要。
- **フォーカス切替**: WASD で銀河を飛び、フォーカス以外の星に `viewDistanceAu` 相当の近距離まで
  近づくと、その星を新フォーカスに再設定（`FloatingOrigin` の原点も付け替え、精度破綻回避）。
  切替判定は「カメラに最も近いカタログ星が現フォーカスでなく、かつ一定 AU 以内」。

## 4. 太陽系データ（本物の 8 惑星）

- `system/solarSystem.ts`: 水金地火木土天海を `Planet[]` でハードコード。
  実軌道長半径（AU）: 0.39/0.72/1.00/1.52/5.20/9.58/19.2/30.1。型: 地=海洋, 金火水=岩石,
  木土=ガス, 天海=氷。相対半径（地球比）: 0.38/0.95/1.0/0.53/11.2/9.45/4.0/3.88。ハビタブルゾーン
  判定は太陽光度=1 で再計算（地球=内、火星=境界外）。`isReal: true`。
- `buildStellarSystem` を拡張し、`index === 0`（太陽）のときは `getSolarSystem()` を返す
  （NASA 実在データにも手続き生成にも該当しないため特別扱い）。太陽の恒星諸元は既存の HYG index 0
  （absmag/ci）から算出。
- 土星の環は `SystemScene` にリング描画を追加（型 or フラグで土星のみ）。

## 5. HUD / UI（直感性）

- 画面下: **速度スライダー**（横長）+ 現在速度表示 + フォーカス天体名（日本語）。
- クリックで天体選択 → 情報パネル（既存の `InfoPanel`/`PlanetPanel` を流用、日本語）。星の場合は
  「ここへ移動」でその星をフォーカスにする動線も可（ボタン、任意）。
- 画面端に半透明の操作ヒント: 「ドラッグ=視点 / ホイール=ズーム / WASD=移動」。
- WebGL2 非対応・カタログ読込失敗の日本語フォールバックは既存を流用。

## 6. アーキテクチャ

- 新規 `nav/NavigationController.ts`: focus・orbit・viewDistanceAu・速度・フォーカス切替を管理する
  純粋寄りの状態機械（Three 非依存の計算部を分離してテスト可能に）。
- 新規 `nav/orbitCamera.ts`: (orbitYaw, orbitPitch, viewDistanceAu, focusPos) → カメラ world 位置・
  注視点 の純粋計算。
- 新規 `nav/fade.ts`: `viewDistanceAu → fadeSystem` の純粋関数。
- 新規 `nav/InputMapper.ts`: ドラッグ/ホイール/WASD/スライダー → NavigationController への意図。
  既存 `InputController` は galaxy 用だったので、この新方式に合わせて置換 or 併存（実装計画で判断）。
- 新規 `system/solarSystem.ts`: 太陽系ハードコードデータ。
- 改修 `src/app.ts`: galaxy/system モード分岐を撤去し、NavigationController + 単一シーン
  （StarField 常時 + フォーカス星の SystemScene を fade 付きで出し入れ）+ 新入力に置換。
- 再利用: `Renderer`（対数深度）, `FloatingOrigin`, `StarField`, `SystemScene`, `PlanetMaterial`,
  `buildStellarSystem`, `planetGen`, `Picker`, `InfoPanel`, `PlanetPanel`, `describeStar`/`describePlanet`。
- 撤去/置換: 現行の `ShipController`（throttle+warp のワープ曲線）と AppMode 分岐、突入/退出ボタン。
  速度は速度スライダー由来のシンプルな並進速度に一本化。

## 8. テスト方針

- vitest 単体テスト:
  - `orbitCamera`: orbit 角 + 距離 + focus → カメラ位置・注視点（既知値・距離不変量）。
  - `fade`: `viewDistanceAu → fadeSystem` の境界（300 AU→1, 30000 AU→0, 単調）。
  - `NavigationController`: WASD 並進・ズーム範囲クランプ・フォーカス切替しきい値（決定論）。
  - `solarSystem`: 8 惑星・軌道昇順・型・相対サイズ・地球がハビタブルゾーン内。
  - `buildStellarSystem`: index 0 → 太陽系データを返す。
- 描画・遷移の滑らかさ・操作感は私（コントローラ）が Playwright で実データ検証。
